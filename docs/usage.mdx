LowCSS adopts a familiar and intuitive utility class naming convention, inspired by [Tailwind CSS](https://www.tailwindcss.com). This syntax allows you to apply a wide range of styles to your HTML elements by adding class attributes directly in your markup.

# Basic Usage

The basic naming pattern for our utility classes is as follows:

```
[{prefix}]{property}-{value}
```

Let's break down each component of the naming pattern:

- `{prefix}` (optional): you can configure LowCSS for adding a custom prefix to all utilities.
- `{property}`: specifies the CSS property that the utility class modifies. Examples of properties include `bg` for background, `text` for text-related styles, `p` for padding, `m` for margin, and so on. This component is not always present and depends on the utility class.
- `{value}`: indicates the specific value or variation for the CSS property. 

For example, to set the text color to red, use the following class:

```html
<p class="text-red-500">Hello, world!</p>
```

This will apply the specified style to the element, resulting in red-colored text.

# Responsive Design

LowCSS makes it easy to create responsive designs using the following utility syntax: 

```
{breakpoint}:[{prefix}]{property}-{value}
```

- `{prefix}`, `{property}`, and `{value}` follow the same structure as mentioned above.
- `{breakpoint}`: represents a specific breakpoint where the utility class should apply. 

The following breakpoints are defined by default in LowCSS:

<table className="w-full text-sm">
    <thead>
        <tr className="border-b border-solid border-gray-200">
            <td className="p-3 font-bold">Breakpoint</td>
            <td className="p-3 font-bold">Minimum width</td>
            <td className="p-3 font-bold">Rule</td>
        </tr>
    </thead>
    <tbody>
        {Object.keys(props.breakpoints).map(key => (
        <tr key={key}>
            <td className="p-3 font-mono">{key}</td>
            <td className="p-3 font-mono">{props.breakpoints[key]}</td>
            <td className="p-3 font-mono">
                <code>@media (min-width: {props.breakpoints[key]}) {"{"} ... {"}"}</code>
            </td>
        </tr>
        ))}
    </tbody>
</table>

You can apply different styles at different breakpoints:

```html
<div class="hidden md:block [...]">
    Content
</div>
```

In ths example, the content text will be hidden at screens sizes lower than the medium breakpoint, and visible is screens sizes upper than the medium breakpoint.

# Pseudo Modifiers

LowCSS includes a range of utility classes for working with pseudo modifiers, allowing you to easily style elements in specific states. We use the following syntax:

```
{pseudo-modifier}:[{prefix}]{property}-{value}
```

## Hover, Focus, and Active

Add the `hover`, `focus` and `active` modifiers to apply utilities only on hover, focus or active.

<ExampleCode className="flex justify-center">
    <div className="bg-blue-500 hover:bg-blue-800 text-white text-sm px-4 py-2 rounded-md cursor-pointer">
        <strong>Hover me</strong>
    </div>
</ExampleCode>

```html
<div class="bg-blue-500 hover:bg-blue-800 [...]">
    Hover me
</div>
```

## Checked, Disabled, and Required

Use `checked`, `disabled` and `required` modifiers to style form elements.

<ExampleCode className="flex items-center justify-center">
    <div className="w-full maxw-xs rounded-md p-6 shadow-sm border border-gray-300">
        <div className="mb-4">
            <div className="font-bold text-sm mb-1">Username (required)</div>
            <input
                type="text"
                className="block p-2 rounded-md border border-solid border-gray-300 required:border-red-500 bg-white w-full text-sm required:text-red-500 outline-none"
                required
            />
        </div>
        <div className="mb-4">
            <div className="font-bold text-sm mb-1">Email (disabled)</div>
            <input
                type="email"
                value="bob@company.com"
                className="block p-2 rounded-md border border-solid border-gray-300 bg-white disabled:bg-gray-200 w-full text-sm"
                disabled
            />
        </div>
        <div className="flex flex-row-reverse">
            <div className="px-4 py-2 bg-blue-500 hover:bg-blue-600 text-white rounded-md text-sm cursor-pointer">
                <strong>Update</strong>
            </div>
        </div>
    </div>
</ExampleCode>

```html
<form>
    <!-- Change background-color when is disabled -->
    <input type="email" disabled className="bg-white disabled:bg-gray-200 [...]" >
</form>
```

## Group Modifiers

You can conditionally style an element based on the state of a parent element. Add a `group` class to the parent element and use `group-hover`, `group-focus` or `group-focus-within` modifiers to style the element.

<ExampleCode className="flex justify-center">
    <div className="group flex items-center justify-center bg-white relative rounded-md py-2 px-4 border border-solid border-gray-300">
        <span className="text-xl text-gray-600">
            <Icon icon="bars" />
        </span>
        <span className="text-gray-600 ml-2">Menu</span>
        <div className="hidden group-hover:block absolute w-48 left-0 top-full pt-1">
            <div className="bg-white border border-gray-300 border-solid rounded-md flex flex-col gap-1 py-1">
                <div className="px-4 py-2 hover:bg-gray-200 cursor-pointer">Dashboard</div>
                <div className="px-4 py-2 hover:bg-gray-200 cursor-pointer">Profile</div>
                <div className="px-4 py-2 hover:bg-gray-200 cursor-pointer">Settings</div>
                <div className="h-0 w-full border-b border-gray-200 border-solid" />
                <div className="px-4 py-2 hover:bg-gray-200 cursor-pointer">Sign out</div>
            </div>
        </div>
    </div>
</ExampleCode>

```html
<div class="group [...]">
    <!-- button content -->
    <div class="hidden group-hover:block [...]">
        <!-- dropdown items -->
    </div>
```
