# Utility class names design

**LowCSS** uses the following utility class name design:

```
[prefix]{class}-{value}
```

- **prefix** (optional): you can provide a prefix for all utilities.
- **class**: is the utility class name (e.g. `text`, `bg`, or `flex`).
- **value**: this allows you to provide the utility value such as text size, color shades, spacing, effects, or filters.

Example:

```html
<div class="text-gray-800 font-bold p-4">
    Hello world
</div>
```

# Pseudo-class variants

Some utilities can be applied conditionally by adding a modifier at the beginning of the utility class that describes the condition that you want to apply. For example, to apply `bg-blue-500` but only on hover, use `hover:bg-blue-500`.


## Hover, focus and active

Add the `hover`, `focus` and `active` modifiers to apply utilities only on hover, focus or active.

<ExampleCode className="flex justify-center">
    <div className="bg-blue-500 hover:bg-blue-800 text-white text-sm px-4 py-2 rounded-md cursor-pointer">
        <strong>Hover me</strong>
    </div>
</ExampleCode>

```html
<div class="bg-blue-500 hover:bg-blue-800 [...]">
    Hover me
</div>
```

## Checked, disabled and required

Use `checked`, `disabled` and `required` modifiers to style form elements.

<ExampleCode className="flex items-center justify-center">
    <div className="p-6 rounded-md bg-white w-full maxw-xs">
        <div className="mb-4">
            <div className="font-bold text-sm mb-1">Username (required)</div>
            <input
                type="text"
                className="block p-2 rounded-md border border-solid border-gray-300 required:b-red-500 bg-white w-full text-sm required:text-red-500 outline-none"
                required
            />
        </div>
        <div className="mb-4">
            <div className="font-bold text-sm mb-1">Email (disabled)</div>
            <input
                type="email"
                value="bob@company.com"
                className="block p-2 rounded-md border border-solid border-gray-300 bg-white disabled:bg-gray-200 w-full text-sm"
                disabled
            />
        </div>
        <div className="flex flex-row-reverse">
            <div className="px-4 py-2 bg-blue-500 hover:bg-blue-600 text-white rounded-md text-sm cursor-pointer">
                <strong>Update</strong>
            </div>
        </div>
    </div>
</ExampleCode>

```html
<form>
    <!-- Change background-color when is disabled -->
    <input type="email" disabled className="bg-white disabled:bg-gray-200 [...]" >
</form>
```

## Group modifiers

You can conditionally style an element based on the state of a parent element. Add a `group` class to the parent element and use `:group-hover` or `:group-focus` to style the element.

<ExampleCode className="flex justify-center">
    <div className="group flex items-center justify-center bg-white relative rounded-md py-2 px-4 border border-solid border-gray-300">
        <span className="text-lg text-gray-600">
            <Icon icon="bars" />
        </span>
        <span className="text-gray-600 ml-2">Menu</span>
        <div className="hidden group-hover:block absolute w-48 left-0 top-full pt-1">
            <div className="bg-white border border-gray-300 border-solid rounded-md flex flex-col gap-1 py-1">
                <div className="px-4 py-2 hover:bg-gray-200 cursor-pointer">Dashboard</div>
                <div className="px-4 py-2 hover:bg-gray-200 cursor-pointer">Profile</div>
                <div className="px-4 py-2 hover:bg-gray-200 cursor-pointer">Settings</div>
                <div className="h-0 w-full border border-gray-200 border-solid" />
                <div className="px-4 py-2 hover:bg-gray-200 cursor-pointer">Sign out</div>
            </div>
        </div>
    </div>
</ExampleCode>

```html
<div class="group [...]">
    <!-- button content -->
    <div class="hidden block:group-hover [...]">
        <!-- dropdown items -->
    </div>
</div>
```

# Responsive

Every utility class can be configured with different breakpoints without using the `@media` rule in your CSS and without leaving your HTML. The following breakpoints are defined by default in LowCSS:

<table className="w-full text-sm">
    <thead>
        <tr className="border-b border-solid border-gray-200">
            <td className="p-3 font-bold">Breakpoint</td>
            <td className="p-3 font-bold">Minimum width</td>
            <td className="p-3 font-bold">Rule</td>
        </tr>
    </thead>
    <tbody>
        {Object.keys(props.breakpoints).map(key => (
        <tr key={key}>
            <td className="p-3 font-mono">{key}</td>
            <td className="p-3 font-mono">{props.breakpoints[key]}</td>
            <td className="p-3 font-mono">
                <code>@media (min-width: {props.breakpoints[key]}) {"{"} ... {"}"}</code>
            </td>
        </tr>
        ))}
    </tbody>
</table>

To conditionally apply a utility at a specific breakpoint, just prefix the utility with the breakpoint name followed by the `:` character:

```html
<!-- Hidden on small screens, visible on medium screens -->
<div class="hidden md:block [...]">
    [...]
</div>
```
