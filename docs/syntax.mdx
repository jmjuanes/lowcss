# Utility class names design

**LowCSS** uses the following utility class name design:

```
[prefix]{class}-{value}
```

- **prefix** (optional): you can provide a prefix for all utilities.
- **class**: is the utility class name (e.g. `text`, `bg`, or `flex`).
- **value**: this allows you to provide the utility value such as text size, color shades, spacing, effects, or filters.

Example:

```html
<div class="text-gray-800 font-bold p-4">
    Hello world
</div>
```

# Pseudo-class variants

Some utilities can be applied conditionally by adding a modifier at the end of the utility class that describes the condition that you want to apply. For example, to apply `bg-blue-500` but only on hover, use `bg-blue-500:hover`.


## Hover, focus and active

Add the `:hover`, `:focus` and `:active` pseudo selectors to apply utilities only on hover, focus or active.

<ExampleCode className="d:flex justify:center">
    <div className="bg:blue-500 bg:blue-800:hover text:white text:sm px:4 py:2 r:md cursor:pointer">
        <strong>Hover me</strong>
    </div>
</ExampleCode>

```html
<div class="bg-blue-500 bg-blue-800:hover [...]">
    Hover me
</div>
```

## Checked, disabled and required

Use `:checked`, `:disabled` and `:required` selectors to style form elements.

<ExampleCode className="d:flex items:center justify:center">
    <div className="p:6 r:md bg:white w:full maxw:xs">
        <div className="mb:4">
            <div className="font:bold text:sm mb:1">Username (required)</div>
            <input
                type="text"
                className="d:block p:2 r:md b:1 b:solid b:gray-300 b:red-500:required bg:white w:full text:sm text:red-500:required outline:none"
                required
            />
        </div>
        <div className="mb:4">
            <div className="font:bold text:sm mb:1">Email (disabled)</div>
            <input
                type="email"
                value="bob@company.com"
                className="d:block p:2 r:md b:1 b:solid b:gray-300 bg:white bg:gray-200:disabled w:full text:sm"
                disabled
            />
        </div>
        <div className="d:flex flex:row-reverse">
            <div className="px:4 py:2 bg:blue-500 bg:blue-600:hover text:white r:md text:sm cursor:pointer">
                <strong>Update</strong>
            </div>
        </div>
    </div>
</ExampleCode>

```html
<form>
    <!-- Change background-color when is disabled -->
    <input type="email" disabled className="bg-white bg-gray-200:disabled [...]" >
</form>
```

## Group modifiers

You can conditionally style an element based on the state of a parent element. Add a `group` class to the parent element and use `:group-hover` or `:group-focus` to style the element.

<ExampleCode className="d:flex justify:center">
    <div className="group d:flex items:center justify:center bg:white position:relative r:md py:2 px:4 b:1 b:solid b:gray-300">
        <span className="text:lg text:gray-600">
            <Icon icon="bars" />
        </span>
        <span className="text:gray-600 ml:2">Menu</span>
        <div className="d:none d:block:group-hover position:absolute w:48 left:0 top:full pt:1">
            <div className="bg:white b:1 b:gray-300 b:solid r:md d:flex flex:col gap:1 py:1">
                <div className="px:4 py:2 bg:gray-200:hover cursor:pointer">Dashboard</div>
                <div className="px:4 py:2 bg:gray-200:hover cursor:pointer">Profile</div>
                <div className="px:4 py:2 bg:gray-200:hover cursor:pointer">Settings</div>
                <div className="h:0 w:full b:1 b:gray-200 b:solid" />
                <div className="px:4 py:2 bg:gray-200:hover cursor:pointer">Sign out</div>
            </div>
        </div>
    </div>
</ExampleCode>

```html
<div class="group [...]">
    <!-- button content -->
    <div class="d-none d-block:group-hover [...]">
        <!-- dropdown items -->
    </div>
</div>
```

# Responsive

Every utility class can be configured with different breakpoints without using the `@media` rule in your CSS and without leaving your HTML. The following breakpoints are defined by default in LowCSS:

<table className="w:full text:sm">
    <thead>
        <tr className="bb:1 bb:solid bb:gray-200">
            <td className="p:3 font:bold">Breakpoint</td>
            <td className="p:3 font:bold">Minimum width</td>
            <td className="p:3 font:bold">Rule</td>
        </tr>
    </thead>
    <tbody>
        {Object.keys(props.breakpoints).map(key => (
        <tr key={key}>
            <td className="p:3 font:mono">{key}</td>
            <td className="p:3 font:mono">{props.breakpoints[key]}</td>
            <td className="p:3 font:mono">
                <code>@media (min-width: {props.breakpoints[key]}) {"{"} ... {"}"}</code>
            </td>
        </tr>
        ))}
    </tbody>
</table>

To conditionally apply a utility at a specific breakpoint, just add a suffix `@` followed by the breakpoint name to the utility class:

```html
<!-- Hidden on small screens, visible on medium screens -->
<div class="d-none d-block@md [...]">
    [...]
</div>
```
