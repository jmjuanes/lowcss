includes a set of CSS variables. LowCSS uses a `@theme` directive to define these variables, which can be used to customize the appearance of your application. {{/prose-paragraph.html}} {{>>prose-code.html language="css" label="main.css" icon="file-code"}} @theme { --color-primary: #1d4ed8; --color-secondary: #f59e0b; } @import "lowcss/utilities.css"; {{/prose-code.html}} {{>>prose-paragraph.html}} These variables are used to generate the utility classes, so based on the previous example, you can use the following classes: {{/prose-paragraph.html}} {{>>prose-code.html language="html" label="index.html" icon="file-code"}}
This div has a primary background and secondary text color.
{{/prose-code.html}} {{>>prose-paragraph.html}} You can also use the `@theme` directive to define your own custom CSS variables, which can be used in your application. This allows you to create a consistent design system that can be easily maintained and updated. {{/prose-paragraph.html}} {{>>prose-code.html language="css" label="components.css" icon="file-code"}} .button { background-color: var(--color-primary); /* ... */ } {{/prose-code.html}} {{>prose-separator.html}} {{>prose-heading.html level="h2" text="Variables Namespaces" id="variables-namespaces"}} {{>>prose-paragraph.html}} Theme variables in LowCSS are organized into namespaces, which helps to map the variables to their respective utility classes. {{/prose-paragraph.html}} {{#each page.attributes.variables}} {{/each}}
