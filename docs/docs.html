This syntax allows you to apply a wide range of styles to your HTML elements by adding class attributes directly in your markup. {{/prose-paragraph.html}} {{>prose-heading.html level="h2" text="Basic Usage" id="usage-basic"}} {{>>prose-paragraph.html}} The basic naming pattern for our utility classes is as follows: {{/prose-paragraph.html}} {{>>prose-code.html language="text"}} {property}-{value} {{/prose-code.html}}
  • {{#inlineMarkdown}}`{property}`: specifies the CSS property that the utility class modifies. Examples of properties include `bg` for background, `text` for text-related styles, `p` for padding, `m` for margin, and so on. This component is not always present and depends on the utility class.{{/inlineMarkdown}}
  • {{#inlineMarkdown}}`{value}`: indicates the specific value or variation for the CSS property.{{/inlineMarkdown}}
{{>>prose-code.html language="html"}} {{/prose-code.html}} {{>prose-heading.html level="h2" text="Media queries" id="usage-media-queries"}} {{>>prose-paragraph.html}} Media queries are a powerful tool for creating responsive designs. LowCSS provides a simple and consistent way to apply styles based on different media conditions. {{/prose-paragraph.html}} {{>prose-heading.html level="h3" text="Responsive Design"}} {{>>prose-paragraph.html}} LowCSS makes it easy to create responsive designs using the following utility syntax: {{/prose-paragraph.html}} {{>>prose-code.html language="text"}} {breakpoint}:{property}-{value} {{/prose-code.html}}
  • {{#inlineMarkdown}}`{property}`, and `{value}` follow the same structure as mentioned above.{{/inlineMarkdown}}
  • {{#inlineMarkdown}}`{breakpoint}`: represents a specific breakpoint where the utility class should apply.{{/inlineMarkdown}}

We use the following syntax: {{/prose-paragraph.html}} {{>>prose-code.html language="text"}} {modifier}:{property}-{value} {{/prose-code.html}} {{>prose-heading.html level="h3" text="Pseudo modifiers"}} {{>>prose-paragraph.html}} Pseudo modifiers allows you to conditionally apply an utility class based on the pseudo state, such as `hover` or `focus`. {{/prose-paragraph.html}}

It helps normalize the default styles applied by different browsers, ensuring a more consistent and predictable rendering of web pages across various devices and browsers. {{/prose-paragraph.html}} {{>>prose-paragraph.html}} For a complete list of changes and styles, please refer to the [source stylesheet](https://github.com/jmjuanes/lowcss/blob/main/reset.css). {{/prose-paragraph.html}} {{>prose-heading.html level="h3" text="Box-sizing"}} {{>>prose-paragraph.html}} It changes the `box-sizing` property of all elements to `border-box`. This ensures that the width and height of elements are calculated including padding and border, providing a more intuitive box model. {{/prose-paragraph.html}} {{>>prose-code.html language="css"}} *, *:before, *:after { box-sizing: border-box; } {{/prose-code.html}} {{>prose-heading.html level="h3" text="Borders are removed in all elements"}} {{>>prose-paragraph.html}} It resets the `border-width` to `0` and initializes the `border-syle` to `solid` in all elements. {{/prose-paragraph.html}} {{>>prose-code.html language="css"}} *, *:before, *:after { border-color: currentColor; border-style: solid; border-width: 0; } {{/prose-code.html}} {{>prose-heading.html level="h3" text="Margins"}} {{>>prose-paragraph.html}} It resets the default margin of some elements (including headings and paragraphs) to `0`. This eliminates any inconsistencies in default spacing applied by different browsers. {{/prose-paragraph.html}} {{>>prose-code.html language="css"}} blockquote, h1, h2, h3, h4, h5, h6, hr, p, pre { margin: 0; } {{/prose-code.html}} {{>prose-heading.html level="h3" text="Headings"}} {{>>prose-paragraph.html}} It normalizes the font-related properties such as `font-family` and `font-weight` in heading elements to provide a consistent typography baseline. {{/prose-paragraph.html}} {{>>prose-code.html language="css"}} h1, h2, h3, h4, h5, h6 { font-size: inherit; font-weight: inherit; } {{/prose-code.html}} {{>prose-heading.html level="h3" text="Text elements"}} {{>>prose-paragraph.html}} Sets the default styles for anchor links elements to remove underline and inherit color: {{/prose-paragraph.html}} {{>>prose-code.html language="css"}} a { color: inherit; text-decoration: inherit; } {{/prose-code.html}} {{>prose-heading.html level="h3" text="Lists"}} {{>>prose-paragraph.html}} It removes the default styles for unordered and ordered lists (`ul` and `ol`) to eliminate any browser-specific list styles. {{/prose-paragraph.html}} {{>>prose-code.html language="css"}} ul, ol, dl { margin: 0; padding: 0; } {{/prose-code.html}} {{>prose-heading.html level="h3" text="Tables"}} {{>>prose-paragraph.html}} It resets the default border-1 styles for tables. {{/prose-paragraph.html}} {{>>prose-code.html language="css"}} table { border-collapse: collapse; border-spacing: 0; } {{/prose-code.html}} {{>prose-heading.html level="h3" text="Form elements"}} {{>>prose-paragraph.html}} It resets the default typography, padding, and margin properties common form elements: {{/prose-paragraph.html}} {{>>prose-code.html language="css"}} button, input, select, textarea { color: inherit; font-family: inherit; font-weight: inherit; line-height: inherit; margin: 0; padding: 0; } {{/prose-code.html}} {{>>prose-paragraph.html}} It also prevents resizing textareas horizontally by default: {{/prose-paragraph.html}} {{>>prose-code.html language="css"}} textarea { resize: vertical; } {{/prose-code.html}} {{/layout-sidebar.html}}
